<meta name="format-detection" content="telephone=no"> - чтобы на Сафари не было 
автоматического преобр-я телефонов на сайте в ссылки, если это не указанно явно

Проще начинать с десктопноый версии, а потом переходить на мобильную
Медиа-запросы должны писаться после всех свойств, чтобы не было их перекрытия.

Порядок медиа-запросов написания от декстом к мобильной верстке и наоборот идет от болшого экрана 
до маленького

напр, от декст до моб-й:
@media(max-width 1199px)
then 
@media(max-width 991px)
then
@media(max-width 767px)
then
@media(max-width 575px)
then
@media(max-width 420/480)

h1 ТОЛЬКО ОДИН на странице. h5, h6 почти никогда не используются.
При смене расположения элементов (опускании формы с инпутами под текст) центрировать текст в инпутах
На адаптиве не должно быть много пустого места и налезания элементов.
Иконочные шрифты подходят, если нужно вставить маленькую иконку в инпут/кнопку
(к прим, изоб-е телефонной трубки, зведздочки)

Если от моб-й до десктопной то наоборот, то есть, идет по возр-ю и вместо max идет min

Секции с описанием обычно шрифтами: 16px, 14px, 12px - большие, средние и малые экраны соотв-но
Все что меньше 12рх нечитаемо!

Обычно заголовки от дизайнеров 60, 54, 52, 50, 48, 42, 36, 28, 24 и иногда 22.
svg - иконки и векторные изоб-я
png - для изоб-й с прозр-м фоном
jpg, jpeg - для фоновых изоб-й и картинок
gif - для анимаций.

img {
    max-width: 100%;
    height: auto;
}
Здесь используется max-width: 100%; для гарантии того, что изображение
никогда не выйдет за пределы ширины родительского контейнера. Если родительский контейнер
сжимается до ширины, меньшей чем ширина  изображения - последнее сжимается вместе с контейнером.
 Установка height: auto; нужна для сохранения пропорций.

От Вадима Макеева:
не пихать все в дивы (диватоз)
использовать section, h1, h2, h3 etc. 
h1 должен быть один на странице (обычно, типа самый большой заголовок)

В теге <picture> выбор картинки идет, используя перебор по srcset сверху вниз:
от верхнего <source> до нижнего,
и если все вар-ты не подошли, подгружается картинка из src от <img>.

Если кликабельный элемент (кнопка, крестик модалки, точки внизу слайдера), то использовать <button>;
если переход между частами страницы(якоря) или на другие тсраницы/сайты, то <a>.

По БЭМу не желательно использовать селекторы по тегу в стилаях (и, соответственно, теги), 
иначе это довляет уникальности элементу и сложно будет использовать его в других проектах.
К прим, нельзя обращаться в сss по id, нужно по классу.

Не задавать фиксированую высоту в блоках с текстом, к примеру, поскольку иначе
блока не будет резиновым и текст будет вылазить за его рамки;
Похожая ситуация может быть с max-height для блоков с текстом
В слайдере реализовать то, что при вставке других изображений человеком
картинки не будут разного размера в самом слайдере

Если под текстом есть какая-то картинка, то должен быть какой-то background
для страховки, на случай, если не подгрузится картинка, то текст можно будет прочитать.

Про семантику вспомнить(исп-е разных тегов, для поисковиков эту гуд, вроде)
В alt для картинок нужно прописівать релевантный текст для поисковиков в целях SEO.
Если крестик сделанн button-ом, то на сафари может появиться лишний левій и правій паддинг

<ul class="list-inline"></ul>(Bootstrap4) - чтобы вс троку шли элементы марк-го списка, 
а не в столбец, напр, нужно для шапки в nav-меню

Задание затемнение через ::before:
1) для род-го блока задать position: relative
2) для ::before:
content: '';
display: block;
position: absolute;
left: 0;
top: 0;
width: 100%;
height: 100%;
background-color: rgba(0, 0, 0, 0.6);
z-index: на единицу меньше, чему у род-го блока;

Для задания анимации для соотв-го блока задаем:
1) animation:   animation-name(orbita) animation-duration animation-timing-function 
		animation-delay 
		animation-iteration-count animation-direction;
2) @keyframes orbita {
0% {свойства}
25% {свойства} и тд до 100%
}
Для воспроизведения анимации достаточно указать только два свойства - 
						animation-name и animation-duration
